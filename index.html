01
C3 Updater — Progress Validation & Update Workflow
Python-based workflow for subcontractor C3 validation, stakeholder routing, tag reconciliation, and EasyPlant update preparation
Python Excel/VBA EasyPlant In Production
Problem

Weekly progress reporting required manually matching subcontractor C3 forms against EasyPlant data — tag by tag, step by step. With multiple subcontractors and thousands of records each, the process took more than 16 hours per week, was difficult to review, and was highly exposed to manual error.

Solution

Desktop application (Python) built as a single workflow for data matching, NA handling, tag reconciliation, delta report generation, and feedback application. The tool prepares upload-ready templates, supports Construction Team review through structured delta files, and closes the loop by applying approved corrections back into the working dataset.

Impact

Reduced weekly reporting effort from 16 hours to 2. Adopted by the Progress Team to streamline update checks, delta review, and upload preparation. The workflow reduced manual comparison work, improved consistency across subcontractor submissions, and made tag-related gaps visible earlier in the reporting cycle.

Coordination

Beyond the tool layer, the workflow operates as a weekly coordination point between subcontractors, Quality department review of NA-status changes, Planning team alignment on progress logic, Construction Team confirm delta, and management reporting. The automation makes that loop scalable: submissions are validated, routed by stakeholder, corrected, and brought back into the working dataset before upload.

03
Engineering Data Validation Pipeline for BI Refresh
Working Python prototype for extracting engineering datasets, validating data quality, and preparing auditable Power BI refresh inputs
Python SQL / ODBC Power BI PowerShell Prototype
Problem

Engineering dashboards often depend on manual exports, inconsistent source files, and refreshes that happen before data quality checks are complete. That creates reporting lag, weakens trust in dashboards, and pushes raw operational data into BI before it is ready for management use.

What I Built

Built a working Python pipeline prototype that can pull engineering datasets either from database tables or from standardized Excel exports, run validation checks, and prepare clean CSV/XLSX outputs for Power BI refresh. The pipeline also generates JSON summary and log files to make refresh runs traceable and auditable.

Validation Scope

Checks include missing GWBS, missing real FWBS values (including placeholder handling), missing drawing references, and incomplete SWC visibility. The aim is not only to refresh dashboards faster, but to create a controlled pre-refresh layer so reporting is fed with cleaner datasets.

Status

Implementation-ready prototype designed for scheduled refresh and adaptation to project-specific schemas, folder structures, and reporting rules.
