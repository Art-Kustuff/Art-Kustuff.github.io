01
C3 Updater — Progress Validation & Update Workflow
Python-based workflow for subcontractor C3 validation, tag reconciliation, and EasyPlant update preparation
Python Excel/VBA EasyPlant Deployed Workflow
Problem

Weekly progress reporting required manually matching subcontractor C3 forms against EasyPlant data — tag by tag, step by step. With 9 subcontractors and thousands of records each, the process took more than 16 hours per week, was difficult to review, and was highly exposed to manual error.

Solution

Desktop application (Python) built as a single workflow for data matching, NA handling, tag reconciliation, delta report generation, and feedback application. The tool prepares upload-ready templates, supports Construction Team review through structured delta files, and closes the loop by applying approved corrections back into the working dataset.

Impact

Reduced weekly reporting effort from 16 hours to 2. Adopted by the Progress Team to streamline update checks, delta review, and upload preparation. The workflow reduced manual comparison work, improved consistency across subcontractor submissions, and made tag-related gaps visible earlier in the reporting cycle.

02
Tag Data Quality Monitoring & Validation Framework
Validation and monitoring logic for tag register completeness, consistency, and reporting readiness
Python Power BI Tag Data Quality Deployed Checks
Problem

Standard system views can show fields as filled while masking placeholder values, incomplete breakdown assignments, and hidden data quality gaps across disciplines. As a result, the reported completeness picture may look stronger than the underlying engineering data actually is.

Scope

Tag Register Validator: checks mandatory fields, placeholders, duplicates, parent references, naming consistency, and status mismatches.

Weekly Health Check: compares snapshots to track improvements, regressions, and completeness change over time.

FWBS Suggestion Logic: generates controlled review suggestions based on existing pattern logic rather than direct automated updates.

Planning Setup Review: checks WC/SWC configuration for quantity without manhours, Current Qty above Target Qty, and negative balance conditions that affect reporting reliability.

Proposed Data Validation Pipeline & Reporting Dashboard
Architecture concept for automated extraction, validation, and reporting refresh
Python SQL Power BI PowerShell Architecture Concept
Architecture Concept

This is a proposed architecture for improving reporting reliability. The idea is to extract tag and configuration data, run validation checks before refresh, and feed cleaner datasets into Power BI so reporting is supported by stronger data controls.

Architecture

Step 1: Python connects to EasyPlant DB via ODBC, pulls Tag Register, Contract Assignment, Planning Setup tables.

Step 2: Validation scripts run: check completeness, detect placeholders, score contractors, flag configuration gaps.

Step 3: Clean datasets saved to shared folder with JSON summary and audit log.

Step 4: Power BI scheduled refresh picks up new data. Dashboard updates automatically.

Step 5: PowerShell Task Scheduler triggers pipeline every morning at 6 AM.
