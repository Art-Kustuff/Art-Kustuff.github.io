01
C3 Updater — Progress Validation & Update Workflow
Python-based workflow for subcontractor C3 validation and EasyPlant update preparation
Python Excel / VBA EasyPlant Deployed Workflow
Problem

Weekly progress reporting required manual comparison of subcontractor C3 submissions against EasyPlant data. In a multi-contractor LNG EPC environment, this was time-consuming and vulnerable to inconsistency and missed deltas.

Solution

Desktop workflow built in Python with automated matching, NA handling, tag reconciliation, delta report generation, and feedback application. The goal was to move from repeated manual comparison to a controlled update-review-correct cycle.

Impact

Reduced weekly reporting effort from 16 hours to 2, improved update consistency, and made tag-related gaps easier to identify before upload preparation.

Workflow Components
▸ Update checks and file matching
▸ Tag reconciliation and upload templates
▸ Delta review workflow for corrections
▸ Feedback application to updated files

Anonymized Engineering Data Quality Dashboard
Portfolio-safe dashboard for tag completeness, configuration readiness, and contractor scoring
Power BI / HTML Demo Tag Data Quality Anonymized Example
Problem

Standard system views can show fields as filled while masking placeholder values, incomplete breakdown assignments, and inconsistent readiness across disciplines or contractors.

Scope

Public-safe demonstration of how tag completeness, FWBS gaps, contractor readiness, and key issues can be surfaced in one engineering data quality view. All metrics and counts in this version were anonymized and adjusted for confidentiality.

Value

Shows how engineering data can be assessed before it reaches dashboards or management reporting, making data quality issues visible earlier and more clearly.
